# CLAUDE.md — 엄태혁의 TIL 레포 운영 가이드

## 프로젝트 정체

이 레포는 엄태혁(GitHub: pandamail)의 개인 학습 아카이브이자 공개 포트폴리오다.

**핵심 원칙**: 내가 정리하면서도, 처음 방문한 사람이 읽었을 때 "이 사람 열심히 했구나"가 느껴져야 한다.

- 단순 메모가 아니라 **읽히는 기록**을 목표로 한다
- README를 통해 폴더 구조를 파악하고, 각 문서로 진입하는 네비게이션 구조를 유지한다
- 주요 기술: Docker, Jenkins, Playwright, Flask, Next.js, Supabase, Claude Code

## 소통 방식

- 항상 한국어로 응답
- 반말로 소통

---

## 레포지토리 구조

```
TIL/
├── 00_Troubleshooting/   # 에러 해결 경험, 원인 분석 기록
├── 01_Infrastructure/    # Docker, Jenkins, Vercel, CLI 환경설정, 배포
├── 02_Backend/           # Flask, Supabase, API, 환경 분리
├── 03_Test_Automation/   # Playwright E2E 자동화 테스트
├── 98_Algorithm/         # 알고리즘 풀이
└── 99_TIL/               # 일별 학습 로그 (99_TIL/YYYY/MM/YYYY-MM-DD-제목.md)
```

### README 네비게이션 원칙

- **루트 README**: 전체 카테고리 테이블 + 주요 프로젝트 + Recent TIL
- **각 폴더 README**: 해당 폴더 내 문서 목록 + 각 문서 1줄 요약 + 링크
- 현재 `01_Infrastructure`만 README 존재. 나머지 폴더는 README 없음 → 작성 기회 있을 때 제안할 것

---

## 문서 작성 기준

### TIL 로그 (99_TIL/)

- 파일명: `YYYY-MM-DD-제목.md`
- 대상 독자: 나(기억 복기) + 처음 보는 개발자 방문객
- 헤더에 날짜, 주제, 핵심 배경 1~2줄 포함
- 회고록: 감정/맥락 포함 OK (이 레포의 특색)
- 강의 노트: 교시/섹션별 정리 + 마지막에 핵심 인사이트 섹션

### 기술 문서 (00~03 폴더)

- 제목만으로 무슨 내용인지 알 수 있어야 함
- 트러블슈팅 문서: 증상 → 원인 → 해결 → 회고 구조
- 아키텍처 문서: 시스템 흐름도(텍스트 또는 Mermaid) 포함 권장
- 코드 스니펫은 언어 태그 필수
