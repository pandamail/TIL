# 📅 2025년 11월 26일 - 일일 학습 기록

## 🚀 오늘 한 일: Jenkins 설치 방식 변경 및 오류 해결

1.  **Jenkins 설치 방식 변경 결정**
    - 기존 Docker 컨테이너 방식으로 운영되던 Jenkins를 리눅스에 직접 설치하는 방식으로 변경하기로 결정했습니다.
    - 안정성과 특정 문제 해결을 위한 선택이었습니다.

2.  **기존 Docker Jenkins 삭제 및 재설치**
    - 기존에 설치되어 있던 Docker Jenkins 컨테이너를 삭제했습니다.
    - 새로운 방식으로 Jenkins를 리눅스 환경에 재설치했습니다.

3.  **플러그인 및 GitLab 연동 재설정**
    - Jenkins 재설치 후 필요한 플러그인들을 다시 설치하고, GitLab과의 연동 설정을 재구성했습니다.

4.  **빌드 성공 및 백엔드 오류 해결**
    - 모든 설정 완료 후 빌드를 시도했으며, 성공적으로 빌드가 완료되었습니다.
    - 이 과정에서 전날부터 발생했던 "백엔드 undefined" 오류가 해결되었음을 확인했습니다.

## 🤔 고민한 부분

- Docker 방식의 Jenkins가 특정 문제(특히 네트워크 또는 환경 설정 관련)에서 예상치 못한 복잡성을 야기할 수 있음을 경험했습니다. 직접 설치 방식이 초기 설정은 더 번거로울 수 있으나, 문제 발생 시 디버깅 및 컨트롤이 용이하다는 장점을 다시 한번 인지하게 되었습니다.
- 문제가 해결되어 다행이지만, 근본적인 Docker Jenkins 문제의 원인을 더 깊게 파악하지 못한 점은 아쉬움으로 남습니다.
