## 핵심 개념 정리

### 1과목 — 컴퓨터 일반

**네트워크 장비 / 프로토콜**
| 장비 | 계층 | 역할 |
|------|------|------|
| 리피터 | 1계층 (물리) | 신호 증폭 |
| 브리지 | 2계층 (데이터링크) | LAN 연결 |
| 라우터 | 3계층 (네트워크) | 최적 경로 설정 |

- 프로토콜 기능: 흐름제어, 동기화, 오류검출, 주소지정
- 단말 자동 인식 = PnP (프로토콜 기능 아님)

**네트워크 토폴로지**
- 링형: 한 노드 고장 시 **전체 장애** 발생 → 신뢰성 낮음
- 망형(Mesh): 특정 노드 고장에도 통신 가능 → 신뢰성 높음

**보안 위협 용어**
| 용어 | 설명 |
|------|------|
| 크래킹 | 시스템 파괴 및 불법 침입 |
| DoS | 서비스 방해 (파괴 없음) |
| 혹스(Hoax) | 악성코드인 척 가장하는 소프트웨어 |

**Windows 세부 동작**
- 인쇄 작업 중인 것도 스풀러에서 강제 종료 가능
- 휴지통 속성: 크기 설정 / 바로삭제 / 확인 대화상자 설정 가능
- 휴지통 최대 크기 초과 시 **자동 비우기 설정은 불가**

### 3과목 — 데이터베이스 일반

**조인 종류**
| 조인 | 설명 |
|------|------|
| 내부 조인 | 양쪽 일치하는 레코드만 |
| 왼쪽 외부 조인 | 왼쪽 전부 + 오른쪽 일치 |
| 오른쪽 외부 조인 | 오른쪽 전부 + 왼쪽 일치 |

**액세스 컨트롤**
- 텍스트 상자: 데이터 표시 + 계산 결과 표시
- 레이블: 고정 텍스트만 (계산 불가)
- 계산 컨트롤 필수 속성 = **컨트롤 원본**
- 레코드 원본 = 폼/보고서 전체에 지정하는 것 (개별 컨트롤 아님)

**보고서 구역**
| 구역 | 출력 시점 |
|------|----------|
| 보고서 머리글 | 시작에 한 번 (로고·제목) |
| 보고서 바닥글 | 마지막에 한 번 |
| 페이지 머리글/바닥글 | 매 페이지마다 |

**SQL 기본**
- 순서: `SELECT → FROM → WHERE → ORDER BY`
- 와일드카드: `*` 여러 문자 / `?` 한 문자 / `#` 숫자 한 자리 / `[!문자]` 해당 문자 제외

**DLookUp 함수 인수 순서**
```
DLookUp("찾을 필드", "테이블", "조건")
```

**데이터 모델 3요소**
- 데이터 구조, 연산, 제약 조건
- 물리적 저장 구조는 포함 안 됨
