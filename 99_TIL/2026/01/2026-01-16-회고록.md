### [SSH 역방향 터널 + socat 최종 구성]

어제 SSH 터널이 `127.0.0.1`에만 바인딩되어 Docker에서 접근이 안 됐던 문제를 socat으로 해결했다.

```
SimMonitor(:8080)
     ↓ SSH 역방향 터널 (-R 18080:localhost:8080)
개발 PC localhost:18080
     ↓ socat (0.0.0.0:18081 → 127.0.0.1:18080)
Docker 컨테이너 (host.docker.internal:18081)
```

```bash
# 태블릿 PC에서 실행
ssh -R 18080:localhost:8080 ibstplan@10.10.61.137

# 개발 PC에서 실행
socat TCP-LISTEN:18081,fork,bind=0.0.0.0,reuseaddr TCP:127.0.0.1:18080 &
```

### [전체 시스템 연동 테스트 성공]

드디어 전체 파이프라인이 정상 동작!

```
LLEAP → 시뮬레이터 → SimMonitor → SIMPREC
```

- CPR, 음성 이벤트, 상태 변화 모두 SIMPREC까지 정상 저장 확인
- `session_events` 테이블에 데이터가 쌓이는 것 확인

### [로그명 불일치 발견]

연동 성공 후 확인해보니 LLEAP 원본 로그명과 SimMonitor 출력 로그명이 다른 것을 발견.

| LLEAP 원본 | SimMonitor 출력 |
|-----------|----------------|
| 콤비튜브 제거 | 발관 |
| 폐음 청진 | 폐 청진 |
| 머리 기울림 / 턱 들어올리기 시행 | 머리 젖히고 턱 들기 |

→ 다음 주에 EventTranslations.cs 대규모 수정 예정
