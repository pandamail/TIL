## 오늘 배운 것

### [Syntax] Python 내장 함수 정리

```python
numbers.sort()              # 오름차순 정렬 (원본 변경)
numbers.sort(reverse=True)  # 내림차순 정렬
sorted(numbers)             # 오름차순 새 리스트 반환

max(numbers)    # 최댓값
min(numbers)    # 최솟값
sum(numbers)    # 합
numbers.count(3)   # 3이 몇 개?
numbers.index(5)   # 5의 위치?

print(*numbers)  # 리스트 언패킹 → 공백 구분 출력
```

### [Structure] 배열 생성 3가지 패턴

```python
# 1. 한 줄 입력
numbers = list(map(int, input().split()))

# 2. 여러 줄 입력 (2차원 배열)
matrix = []
for _ in range(N):
    row = list(map(int, input().split()))
    matrix.append(row)

# 3. 빈 배열 미리 만들기
matrix = [[0] * N for _ in range(N)]
```

---

### [Logic] 2차원 배열 탐색 — 가로/세로

```python
# 가로 탐색
for row in matrix:
    for num in row:
        ...

# 세로 탐색
for col in range(N):
    for row in range(N):
        num = matrix[row][col]
        ...
```

1979번 핵심: 연속된 1의 카운트는 `else` 블록에서 리셋하고, **줄 끝에서 한 번 더 체크**해야 한다.

```python
for num in row:
    if num == 1:
        count += 1
    else:
        if count == K:
            answer += 1
        count = 0
if count == K:  # ← 이게 없으면 마지막 구간 놓침
    answer += 1
```

---

### [실수 기록] 자주 틀렸던 부분

- `short(j)` → `short[j]` : 배열 접근은 `[]`, 함수 호출은 `()`
- `count = 0`의 위치: for문 **밖**에 두면 매번 리셋됨 → `else` 블록 안에서만 리셋

## 보류한 문제: 1928 Base64 Decoder

base64 라이브러리를 써서 풀었는데 SWEA는 표준 라이브러리 외에 제출이 막혀 있어서 통과 못 했다. 라이브러리 없이 직접 디코딩 로직을 짜야 하는데 지금 시점에서는 시간 대비 효율이 낮다고 판단해 보류. D-4에 억지로 붙잡고 있기엔 너무 복잡한 문제였다.
