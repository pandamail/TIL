## 오늘 배운 것

### [Syntax] Python엔 `char` 타입이 없다

Java에서 넘어온 습관으로 `char`를 찾거나 `/* */` 주석을 쓰려다 막혔다.  
Python에서 문자열은 그냥 `str`이고, 문자 하나도 길이 1짜리 `str`이다.

```python
s = str(i)
clap = s.count('3') + s.count('6') + s.count('9')  # count()로 바로 셈
print('-' * clap)  # 문자열 곱셈으로 '-' 반복 출력
```

`s.count()`, `s[::-1]` — 파이썬은 문자열을 다루는 내장 도구가 강력하다. 자바식으로 루프 돌릴 생각부터 접자.

---

### [Logic] 격자 탐색의 공식: `range(start, start + length)`와 `N - M + 1`

파리 퇴치 문제(2001)에서 가장 헷갈렸던 부분이다.

```
N×N 격자에서 M×M 파리채를 움직인다고 할 때:
- 시작점 범위: range(N - M + 1)  → 파리채가 격자 밖으로 나가지 않도록
- 탐색 범위:   range(i, i + M)   → 시작점에서 크기만큼 훑음
```

예) N=5, M=2이면 시작점은 0,1,2,3까지 가능 → `range(5 - 2 + 1)` = `range(4)`.  
`range(i, i + M)`은 i에서 M칸을 훑는다는 뜻. `range(i, M + i)`라고 써도 같다.

---

### [Structure] 2차원 배열 정복

```python
matrix = []
for _ in range(N):
    row = list(map(int, input().split()))
    matrix.append(row)
```

`matrix[x][y]`로 접근. (i, j)는 파리채 **시작점**, (x, y)는 파리채 **내부 탐색점**이라는 역할 분담을 머릿속에 고정해야 한다.  
초반에 `matrix[i][j]`를 4중 루프 안에서 중복 덧셈하는 실수를 했다.

---

### [Efficiency] 4중 루프 vs 슬라이싱

오늘은 4중 루프로 통과시켰지만, 파이썬답게 쓰면 이렇게 줄일 수 있다:

```python
# 4중 루프 버전 (오늘 작성)
for x in range(i, i + M):
    for y in range(j, j + M):
        current_sum += matrix[x][y]

# 파이썬다운 버전
current_sum = sum(matrix[x][y] for x in range(i, i+M) for y in range(j, j+M))
```

정답을 먼저 맞추고, 그다음에 줄이자. 순서가 중요하다.
