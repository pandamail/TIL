## I. 앙상블 기법이란

하나의 데이터를 활용해서 여러 모델을 독립적으로 학습시키고, 각 모델의 예측값을 종합해 최종 결론을 내리는 기법이다. 핵심 아이디어는 **"혼자 틀려도 모이면 맞다"** — 개별 모델의 오차가 서로 상쇄되면서 분산(Variance)이 줄어든다.

앙상블은 학습 방향에 따라 두 계열로 나뉜다.

| 구분 | 방식 | 대표 알고리즘 |
|------|------|--------------|
| **Bagging** | 병렬(Parallel) — 독립적으로 동시 학습 | Random Forest |
| **Boosting** | 순차(Sequential) — 이전 모델의 오차를 다음 모델이 보완 | Adaboost, GBM, XGBoost, LightGBM, CatBoost |

## II. 알고리즘별 특징 정리

### Bagging — Random Forest (2001)

- 전체 데이터에서 랜덤 샘플링 + 랜덤 Feature 추출 → 여러 Decision Tree를 병렬 학습
- 최종 예측은 다수결 원칙(Majority Voting)
- 단일 Decision Tree의 고질적 문제인 Overfitting(과적합)을 구조적으로 해소

### Boosting — Adaboost (Adaptive Boosting)

- 오분류된 데이터에 가중치를 높여 다음 모델이 집중적으로 학습하도록 유도
- 모든 오분류 샘플에 동일한 방식으로 가중치를 부여 (오차의 크기는 구분하지 않음)
- 여러 약한 분류기(weak classifier)를 가중합하여 최종 예측

### Boosting — GBM (Gradient Boosting Machine, 1999)

- Adaboost에서 한 걸음 더 나아가, 단순 가중치가 아닌 **오차의 크기(Gradient)** 로 다음 모델의 학습 방향을 결정
- 오차의 방향(+/-)까지 반영하므로 오분류를 더 세밀하게 보정
- 예측 성능은 높지만 학습 속도가 느리고 과적합에 취약한 단점이 있음

### Boosting — XGBoost (2016)

- GBM의 단점을 보완: 병렬처리 + 정규화 + 균형 트리 분할(깊이 최소화)
- 대용량 데이터에서도 GBM 대비 훨씬 빠른 학습

### Boosting — LightGBM (2017)

- XGBoost보다 더 빠른 속도를 목표로 설계
- **리프 중심(Leaf-wise) 분할**: 모든 리프를 균등하게 키우는 대신 Gain이 가장 큰 리프 노드만 선택적으로 확장
- 중요한 인스턴스·변수만 추려서 트리를 구성하므로 메모리 효율도 높음

### Boosting — CatBoost (2017)

- **범주형(Category) 변수가 많은 데이터**에서 특히 강점
- XGBoost 대비 약 8배 빠르고 불균형 데이터(Imbalanced Data)에도 강함
- **대칭 트리(Oblivious Tree)** 분할: 완전 이진 트리를 생성하여 일관된 구조 유지

## III. 실습#1 — 통신사 고객 이탈 예측 모형 (Churn_JOIN.csv)

- 목표변수: `Churn` (이탈 여부, Y/N)
- 독립변수: 명목형 + 수치형 혼합
- 분할 비율: Train 70% / Test 30%

|-------|----------|-----------|--------|----|

## IV. 변수 선택 (Feature Selection)

### 래퍼(Wrapper) 방식

- 실제 모델을 학습하고 성능을 평가하면서 변수 조합을 탐색
- 변수의 통계적 특성보다 모델 예측 성능 향상 여부에 초점
- 계산 비용은 높지만 현실적 최적 변수 조합을 찾는 데 효과적

### 역방향 제거 (Backward Elimination)

- 모든 변수를 포함한 모델로 시작
- 중요도 낮은 변수부터 단계적으로 제거
- 제거 후 성능 평가 반복 → 최적 변수 조합 탐색

### 변수 중요도 평가 — Gain Ratio

- 엔트로피 감소량 기준으로 변수의 구분 능력을 정량화
- `Gain Ratio(X) = InfoGain(X) / SplitInfo(X)`

### 실습#2 결과 — 역방향 제거 적용

전체 25개 변수에서 단계적으로 제거하며 성능을 관찰한 결과, Step 7(18개 변수)에서 성능 피크에 도달했다.

```
Accuracy 최고: 0.692 (Step 7, F18 제거 시점)
F1 최고      : 0.690
→ Step 9 이후(Gender, Sduration 제거)부터 성능 급락
```

변수가 많다고 무조건 좋은 게 아니다. 불필요한 변수는 노이즈 역할을 해서 오히려 성능을 낮춘다.

## V. 언어 모델의 발전

```
RNN(1986) → LSTM(1997) → Seq2Seq(2014) → Attention(2015)
→ Transformer(2017) → GPT-1(2018) → BERT(2019) → GPT-3(2020)
```

전환점은 Transformer(2017)다.

- **이전**: 고정된 크기의 context vector로 정보를 압축 → 긴 문장에서 정보 손실 발생
- **이후**: 입력 시퀀스 전체를 동시에 참조하는 Attention 메커니즘 → 단어 간 거리에 무관하게 관계 포착

## VI. 실습#3 — BERT Fine-tuning 기반 아마존 리뷰 감성 분류

### Fine-tuning 전체 프로세스

```
[1단계] 하이퍼파라미터 튜닝
Train (90%) + Validation (10%)
→ Pre-trained BERT에 epoch 1~10 돌리며 Loss/Accuracy 관찰
→ Validation Loss 최저점 = 최적 epoch 도출

[2단계] Fine-tuning
최적 하이퍼파라미터로 전체 Train 데이터 학습
→ BERT 가중치 업데이트

[3단계] 성능평가
Test Data → 파인튜닝된 BERT 모델 → 최종 성능 측정
```

### 주요 하이퍼파라미터

| 파라미터 | 설명 | 대표 값 |
|----------|------|---------|
| Learning rate | 가중치 업데이트 보폭 크기 | 2e-5, 3e-5, 5e-5 |
| Batch size | 한 번에 학습하는 샘플 수 | 16, 32 |
| Epochs | 전체 데이터 반복 횟수 | 2~4 |
| Optimizer | 가중치 업데이트 알고리즘 | AdamW |
| Dropout | 과적합 방지용 뉴런 비활성화 비율 | 0.1 |
| Warmup Steps | 학습 초기 학습률 점진적 증가 구간 | 전체 스텝의 10% |
| Weight Decay | L2 정규화로 과적합 방지 | 0.01 |
| Max Seq Length | 입력 텍스트 최대 토큰 길이 | 128, 256, 512 |

### 하이퍼파라미터 튜닝 판단 기준

```
Train Loss 감소 + Validation Loss 감소 → 정상 학습 중
Train Loss 감소 + Validation Loss 상승 → 과적합 시작
→ Validation Loss 최저점 = 최적 epoch
```

### 실습#3 결과

| Model | Accuracy | Precision | Recall | F1 |
|-------|----------|-----------|--------|----|
| BERT (Fine-tuned) | **0.955** | **0.955** | **0.955** | **0.955** |

앙상블 모형(GBM 0.778) 대비 약 18%p 추가 향상. 비정형 텍스트 분류에서는 사전학습된 언어 표현이 압도적으로 유리하다.

## VII. AI 프롬프트 구조화 6단계 프레임워크

프롬프트의 품질이 결과물의 품질을 결정한다.

```
1. 데이터 위치 — 파일 경로 또는 데이터 구조 명시
2. 목표변수 / 독립변수 — 분석 대상 명확히
3. 독립변수 구성 설명 — 명목형/수치형 등 타입 포함
4. 분할 비율 + 사용 모델 — 재현 가능하도록 구체화
5. 평가 지표 — Accuracy, F1, ROC-AUC 등
6. 출력 형태 — 시각화 여부, 표 형식 등
```

**연속 프롬프트 활용 팁:**

```
"위 결과에서 가장 accuracy가 높은 모델의
혼동행렬과 ROC를 시각화하는 코드를 추가해줘"
```

앞 대화의 맥락을 이어받아 최고 모델을 자동으로 특정하기 때문에 모델명을 직접 언급하지 않아도 된다. 컨텍스트를 누적하는 AI일수록 이런 연속 프롬프트 전략이 효과적이다.

## 오늘의 핵심 인사이트

1. **성능은 계단식으로 향상된다** — 단일 모형(0.638) → 앙상블(0.778) → BERT Fine-tuning(0.955). 데이터의 성격(정형/비정형)에 맞는 모델 선택이 성능 점프의 핵심이다.
2. **변수는 줄이는 게 이득일 수 있다** — 25개 → 18개로 줄였을 때 오히려 성능이 올랐다. 불필요한 변수는 신호가 아닌 노이즈다.
3. **Boosting 계열의 진화 방향** — GBM → XGBoost(속도·정규화) → LightGBM(리프 분할·경량화) → CatBoost(범주형·불균형 특화). 데이터 특성에 맞는 모델을 고르는 안목이 실무 역량이다.
4. **BERT의 강점은 사전학습된 언어 표현** — 텍스트 분류에서 피처 엔지니어링 없이도 0.955를 찍는다. Fine-tuning은 전이학습의 가장 실용적인 형태다.
5. **하이퍼파라미터 튜닝에는 정답이 없다** — Validation Loss 최저점을 기준으로 실험 기록을 남기는 습관이 쌓이면 직관이 생긴다.
