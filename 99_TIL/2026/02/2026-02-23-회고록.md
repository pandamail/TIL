# 2026년 2월 23일 (월) - Generic 파라미터 처리 및 CPR 이벤트 감지 구현

## 🎯 주요 성과
- QCPR 하드코딩 5개 if/else 체인을 범용 Laerdal.Response.* 핸들러로 교체
- QCPR 한글 번역 16개 → 31개 확장 + 콤마 숫자 포맷 + 단위 표시
- CPR 이벤트(압박 시작/중단, CPR 시작/중지) 감지 패턴 추가
- 전체 이벤트 타입에 UUID 전달 완료

## 📋 작업 내용

### 1. Generic ParameterValues 처리 (`84c8c0f`)

- **WcfEventCapture.cs**: ProcessLogElement 내 5개 하드코딩된 QCPR if/else 체인을 모든 `Laerdal.Response.*` 파라미터 범용 핸들러로 교체
- **Services/MonitorService.cs**: `[WCF Vitals]` 로그를 key=value 형식으로 변경
- **Services/EventTypeMapper.cs**: QCPR/CLINICAL 명시적 매핑 추가

### 2. QCPR 한글 번역 + 콤마 숫자 + 단위 (`3b3ef03`)

- **ParameterTranslations.cs**: QCPR 번역 16→31개 확장, `ExtractParamUnit()`, `FormatUnit()`, `FormatValueWithUnit()` 공용 메서드 추가
- **Services/MonitorService.cs**: `[WCF Vitals]` 로그에 한글 번역 + 콤마 포맷 + 단위 적용
- **WcfEventCapture.cs**: 로컬 `ExtractParamUnit` 제거, 공용 메서드 사용으로 전환

### 3. QCPR → SIMPREC HTTP API 전송 활성화 (`94fffb2`)

- **Services/MonitorService.cs**: `IsSignificantParameter()`에 QCPR 체크 추가
- QCPR 파라미터도 HTTP API를 통해 SIMPREC으로 전송되도록 활성화

### 4. CPR 이벤트 감지 (`73e800a`) ⭐

- **WcfEventCapture.cs**: `CompressedParameterList` 내 개별 `GString(<LogElement>)` 감지하는 **패턴 B** 추가
  - 기존 패턴 A는 `LogElementsType` 래퍼만 처리
  - 패턴 B로 압박 시작/중단, CPR 시작/중지 이벤트 캡처 가능
- **WcfEventCapture.cs**: `ProcessLogElement`에 `<Event EventId="Laerdal.Event.xxx">` 태그 파싱 + 한글 번역 + `RaiseEvent` 추가

### 5. HTTP API에 UUID 전달 (`e9f93cd`)

- **SimprecClient.cs**: `EventRequest`에 `uuid` 필드 추가, `SendEventAsync`에 uuid 파라미터 추가
- **Services/MonitorService.cs**: `SendEventAsync` 호출 시 `e.Uuid` 전달 (3곳)

### 6. ProcessLogElement `<Event>` 중복 방지 (`5ec3cbf`)

- **WcfEventCapture.cs**: `_eventOccurredTimes` 딕셔너리 추가
  - `ParseEventOccurred`에서 발생한 이벤트를 기록
  - `ProcessLogElement`의 `<Event>` 핸들러에서 2초 이내 동일 이벤트 억제
- 일반 이벤트(Call for help 등)는 `ParseEventOccurred`에서 1회만, CPR 이벤트는 `ProcessLogElement`에서만 통과

### 7. 모든 이벤트 타입에 UUID 전달 (`0fc3ff8`)

- **WcfEventCapture.cs**: DRUG, THEME, STATE, TREND, SOUND의 `RaiseEvent` 호출에 `uuid: _currentBlockUuid` 추가
- 기존 SESSION/EVENT만 전달하던 것을 전체 이벤트 타입으로 확대

## 💡 핵심 교훈
1. **하드코딩 → 범용화**: 5개 if/else를 범용 핸들러로 바꾸면서 향후 새 파라미터 추가 시 코드 수정 불필요
2. **공용 메서드 추출**: `ExtractParamUnit`, `FormatUnit` 등을 공용으로 빼면서 WcfEventCapture와 MonitorService 간 로직 중복 제거
3. **중복 이벤트 전략**: 경로별(ParseEventOccurred vs ProcessLogElement) 역할 분리 + 시간 기반 억제로 깔끔한 dedup 구현

## 🤔 회고
오늘은 SimMonitor의 이벤트 처리 파이프라인을 대폭 개선한 날이다. 가장 의미 있는 작업은 QCPR 하드코딩을 범용 핸들러로 교체한 것과 CPR 이벤트 감지를 위한 패턴 B 추가다. 특히 CompressedParameterList 안에 개별 GString으로 들어오는 LogElement를 잡아내는 패턴 B는, 기존 패턴 A만으로는 놓치던 CPR 시작/중지 이벤트를 캡처할 수 있게 해주었다. 7개 커밋을 체계적으로 쌓아가며 하나씩 문제를 해결한 것이 뿌듯하다. 내일은 파라미터 변경 이벤트 중복 전송 문제를 마무리하면 이벤트 파이프라인 정비가 완성된다.
